## Кооп-режим "Drop-in герой" (Вариант 4)

**Статус:** Планируется

### Концепция
Друг подключается как второй полноценный герой через браузер. Хост авторитетный — управляет миром, гость отправляет только input.

### Архитектура
```
Гость (браузер) --input--> WebSocket Relay (Node.js на Synology) --input--> Хост (браузер)
Гость <--game state-- Relay <--game state-- Хост
```

### Задачи

#### Сервер (~140 строк)
- [ ] `server/relay.js` — WebSocket relay, комнаты, коды подключения
- [ ] `server/Dockerfile` — Node.js контейнер
- [ ] Обновить `docker-compose.yml` — добавить сервис relay
- [ ] Настроить nginx reverse proxy `/ws` -> relay

#### Клиент — новые файлы (~400 строк)
- [ ] `js/network.js` — WebSocket клиент, сериализация state, отправка input
- [ ] `js/lobby.js` — UI "Создать комнату" / "Ввести код"

#### Клиент — доработки (~540 строк)
- [ ] `js/main.js` — второй player, разделить gameLoop на update/render, отправка state
- [ ] `js/combat.js` — player2 атакует врагов
- [ ] `js/enemies.js` — враги агрятся на ближайшего из двух
- [ ] `js/bosses.js` — босс атакует обоих
- [ ] `js/camera.js` — камера на середину между игроками
- [ ] `js/input.js` — разделить локальный и сетевой input
- [ ] `js/dialog.js` — не ставить паузу в кооп
- [ ] `js/save.js` — гость сохраняет прогресс отдельно
- [ ] UI — кнопка "Кооп" в меню, HUD для второго игрока

#### Сложные моменты
- Сериализация game state (что отправлять часто, что редко)
- Интерполяция позиции гостя (сглаживание лага)
- Синхронизация порталов (смена карты)
- Камера при разбежавшихся игроках
- Обработка дисконнекта

### Оценка: ~1100 строк, 3-4 сессии

## Бесконечный открытый мир (чанки)

**Статус:** Планируется

### Концепция
Одна огромная бесшовная карта. Мир генерируется чанками (участками) по мере движения игрока. Каждый чанк — уникальный биом со своими врагами, ресурсами, структурами.

### Архитектура
```
Игрок двигается → проверка: есть ли чанк впереди? → нет → генерируем из seed
                                                    → да → загружаем из кэша
```

- Мир бесконечен, но детерминирован (seed-based) — один и тот же seed = один мир
- Чанки 30x20 тайлов (размер текущей карты)
- Вокруг игрока загружены 3x3 чанков (текущий + 8 соседних)
- Далёкие чанки выгружаются из памяти

### Биомы

| Биом | Тайлы | Враги | Структуры |
|------|-------|-------|-----------|
| Равнина | Трава, дорожки | Слаймы, волки | Деревни (NPC, торговцы) |
| Лес | Деревья, поляны | Волки, бандиты | Лагеря бандитов, сундуки |
| Горы | Камни, ущелья | Скелеты, големы | Пещеры (вход в подземелье) |
| Пустошь | Песок, лава | Огненные враги | Руины, алтари |
| Болото | Вода, грязь | Ядовитые слаймы | Хижины ведьм |
| Снег | Снег, лёд | Ледяные големы | Замёрзшие крепости |

### Задачи

#### Генератор мира (~400 строк)
- [ ] `js/worldgen.js` — seed-based генерация чанков
- [ ] Шумовые функции (Simplex/Perlin) для биомов, высот, рек
- [ ] Таблица биомов по высоте + влажности
- [ ] Размещение структур (деревни, лагеря, пещеры)

#### Система чанков (~300 строк)
- [ ] `js/chunks.js` — загрузка/выгрузка, кэш, бесшовные стыки
- [ ] Переход между чанками без порталов (бесшовный скроллинг)
- [ ] Спавн врагов при генерации чанка

#### Доработки (~200 строк)
- [ ] `js/main.js` — заменить loadMap на chunk-систему для режима "открытый мир"
- [ ] `js/camera.js` — рендер нескольких чанков одновременно
- [ ] `js/tilemap.js` — поддержка multi-chunk рендера
- [ ] Миникарта — показать исследованные чанки

#### Сохранение
- [ ] Сохранять seed мира + список изменённых чанков (убитые враги, открытые сундуки)
- [ ] При загрузке — регенерировать из seed, применить изменения

### Сложные моменты
- Стыковка границ чанков (враги/тайлы не должны "резаться")
- Производительность — не рендерить то, что за экраном
- Баланс — враги должны усиливаться с удалением от центра мира
- Как совместить с сюжетными локациями (деревня, замок, боссы)

### Оценка: ~900 строк, 3-4 сессии

## Рандомные события на картах

**Статус:** Планируется

### Концепция
Существующие карты остаются, но при каждом посещении появляются случайные элементы: дополнительные враги, сундуки с лутом, мини-боссы, ловушки, NPC-путники.

### События

| Событие | Шанс | Где | Что происходит |
|---------|------|-----|----------------|
| Сундук с лутом | 30% | Любая боевая | Рандомное оружие/зелье/монеты |
| Элитный враг | 20% | Любая боевая | Усиленный враг (x2 HP/ATK), гарантированный дроп |
| Торговец-путник | 15% | Любая | Рандомный NPC с редкими товарами |
| Засада | 15% | Лес, ущелье | Группа бандитов появляется вокруг игрока |
| Мини-босс | 10% | Подземелья | Промежуточный босс между этажами |
| Артефакт | 5% | Любая боевая | Временный бафф (x2 ATK, бессмертие 30 сек) |
| Портал в секретку | 5% | Любая | Секретная комната с лутом и сильными врагами |

### Задачи

#### Система событий (~250 строк)
- [ ] `js/events.js` — генератор событий, таблица вероятностей, привязка к карте
- [ ] Спавн сундуков, элитных врагов, NPC при загрузке карты
- [ ] Временные баффы (таймер, визуальный эффект)

#### Доработки (~150 строк)
- [ ] `js/main.js` — вызов генератора событий в loadMap
- [ ] `js/enemies.js` — элитные враги (префикс, свечение, x2 стат)
- [ ] `js/npc.js` — рандомный торговец-путник (случайные товары)
- [ ] Визуал сундуков в `js/sprites.js`

### Оценка: ~400 строк, 1-2 сессии

## Генерация квестов

**Статус:** Планируется

### Концепция
Помимо 10 ручных квестов, NPC выдают рандомные задания. Новые квесты появляются после выполнения предыдущих. Бесконечный контент.

### Типы генерируемых квестов

| Тип | Шаблон | Пример |
|-----|--------|--------|
| Охота | "Убей N [врагов] в [локации]" | "Убей 5 волков в Тёмном лесу" |
| Сбор | "Найди N [предметов]" | "Собери 3 зуба скелета" |
| Доставка | "Отнеси [предмет] из [A] в [B]" | "Отнеси письмо из деревни в замок" |
| Зачистка | "Зачисти [локацию]" | "Уничтожь всех бандитов в ущелье" |
| Арена | "Продержись N раундов" | "Пройди 10 раундов на арене" |
| Эскорт | "Проведи [NPC] до [локации]" | "Доведи торговца до деревни" |
| Босс-рематч | "Победи [босса] заново" | "Убей Лесного стража ещё раз" |

### Система наград

| Сложность | Монеты | XP | Доп. награда |
|-----------|--------|----|----|
| Лёгкий | 20-50 | 15-30 | Зелье |
| Средний | 50-150 | 30-80 | Рандомное оружие |
| Сложный | 150-500 | 80-200 | Редкая броня |

### Задачи

#### Генератор квестов (~300 строк)
- [ ] `js/questgen.js` — шаблоны, генерация из таблиц, рандомизация
- [ ] Привязка к текущему уровню игрока (масштабирование сложности)
- [ ] Уникальные названия квестов (комбинации прилагательных + существительных)

#### Доработки (~200 строк)
- [ ] `js/quests.js` — поддержка динамических квестов наряду с ручными
- [ ] `js/npc.js` — NPC выдают рандомные квесты (Старейшина, Король)
- [ ] `js/main.js` — рандомные квесты обновляются при выполнении
- [ ] UI квестов — различать ручные (золотые) и рандомные (серебряные)

### Оценка: ~500 строк, 1-2 сессии
