## Открытый мир — Прогресс

### ✅ Сессия 1 — Ядро (2026-04-10)
- Simplex noise генерация (`js/simplex.js`)
- 6 биомов: равнина, лес, горы, пустошь, болото, снег
- 7 новых типов тайлов (песок, снег, грязь, камни, мёртвое дерево, снежное дерево, болотная вода)
- Чанковая система 30x20 тайлов с 3x3 загрузкой (`js/chunks.js`, `js/worldgen.js`)
- Бесшовный multi-chunk рендеринг (`js/tilemap.js`)
- Камера без границ для бесконечного мира (`js/camera.js`)
- Масштабирование сложности врагов по расстоянию от центра (5 тиров)
- Сохранение/загрузка seed + изменений мира (`js/save.js`)
- Портал из деревни (север) в открытый мир
- Map proxy для совместимости AI врагов

### ✅ Сессия 2 — Структуры (2026-04-10)
- 6 типов структур (`js/structures.js`):
  - Деревни NPC (8x6) — торговец + целитель + чекпоинт
  - Лагеря бандитов (6x6) — 3-5 бандитов + сундук
  - Входы в пещеры (3x3) — портал в подземелье
  - Руины (8x8) — элитные враги + 2 сундука + алтарь
  - Хижины ведьм (4x4) — NPC-ведьма + зелья
  - Крепости (10x10) — мини-босс + сундуки
- Процедурное размещение по шуму с привязкой к биомам
- NPC с диалогами в открытом мире (странствующий торговец, целитель, ведьма)
- Сундуки и баффы в структурах

### ✅ Сессия 3 — Миникарта (2026-04-10)
- Расширенная миникарта 120x120 (`js/minimap.js`)
- Биомы отображаются цветом на уровне чанков
- Туман войны — неисследованные чанки серые
- Иконки структур (цветные точки по типу)
- "?" маркеры на границе исследованной территории
- Пульсирующая точка игрока
- Красные точки врагов
- Кэширование для производительности

### ✅ Сессия 4 — Быстрое перемещение + Сложность (2026-04-10)
- Быстрое перемещение (`js/fasttravel.js`):
  - Клавиша T для открытия меню
  - Телепортация между посещёнными структурами
  - Деревня Брайтхолл всегда доступна
  - Навигация ↑↓, Enter, Escape
- Настройки сложности (`js/difficulty.js`):
  - 3 пресета: Легко (x0.7), Нормально (x1), Хардкор (x1.5)
  - Клавиша D для переключения
  - Множители HP/ATK врагов + количество
  - Отображение в HUD

### ✅ Сессия 5 — Опасности биомов + Мировые события (2026-04-10)
- Опасности биомов (`js/biome-hazards.js`):
  - Болото: яд (2 HP/сек на грязи у ядовитой воды)
  - Снег: замедление 30% + урон после 8 сек
  - Пустошь: огненные гейзеры (15 урона каждые 3-5 сек)
  - Горы: камнепады (10 урона каждые 4-6 сек)
  - Визуальные эффекты: зелёный/синий оверлей, предупреждения
- Мировые события (`js/world-events.js`):
  - Нашествие (красный): 8-12 врагов, XP x2
  - Караван (жёлтый): редкий лут по сниженным ценам
  - Метеорит (фиолетовый): уникальное оружие/монеты
  - Портал хаоса (тёмно-красный): 3 волны врагов, усиленный лут
  - Появляются каждые 3-5 минут, маркеры на миникарте
  - Визуальные лучи света в игровом мире

### ✅ Сессия 6 — Боссы в биомах + Портал возврата (2026-04-10)
- 4 логова боссов с детерминированным размещением по seed:
  - `boss_lair_forest` — Лесной страж (лес, 2-3 чанка)
  - `boss_lair_wasteland` — Огненный дракон (пустошь, 4-5 чанков)
  - `boss_lair_snow` — Ледяной лич (снег, 6-8 чанков)
  - `boss_lair_dark` — Тёмный маг (любой биом, 13-16 чанков)
- `computeBossLairPositions()` в `structures.js` — детерминированный подбор чанков
- Village portal на чанке (0,0) — возврат в деревню через портал или fast travel
- Спавн настоящих мультифазных боссов (`createBoss`) при входе в чанк с логовом
- Трекинг убитых боссов в save/load (`_killedOpenWorldBosses`)
- Масштабирование HP/ATK боссов по настройке сложности
- Иконки боссов на миникарте с пульсацией
- Fast travel в деревню теперь вызывает `exitOpenWorld()`

## Кооп-режим "Drop-in герой" (Вариант 4)

**Статус:** Планируется

### Концепция
Друг подключается как второй полноценный герой через браузер. Хост авторитетный — управляет миром, гость отправляет только input.

### Архитектура
```
Гость (браузер) --input--> WebSocket Relay (Node.js на Synology) --input--> Хост (браузер)
Гость <--game state-- Relay <--game state-- Хост
```

### Задачи

#### Сервер (~140 строк)
- [ ] `server/relay.js` — WebSocket relay, комнаты, коды подключения
- [ ] `server/Dockerfile` — Node.js контейнер
- [ ] Обновить `docker-compose.yml` — добавить сервис relay
- [ ] Настроить nginx reverse proxy `/ws` -> relay

#### Клиент — новые файлы (~400 строк)
- [ ] `js/network.js` — WebSocket клиент, сериализация state, отправка input
- [ ] `js/lobby.js` — UI "Создать комнату" / "Ввести код"

#### Клиент — доработки (~540 строк)
- [ ] `js/main.js` — второй player, разделить gameLoop на update/render, отправка state
- [ ] `js/combat.js` — player2 атакует врагов
- [ ] `js/enemies.js` — враги агрятся на ближайшего из двух
- [ ] `js/bosses.js` — босс атакует обоих
- [ ] `js/camera.js` — камера на середину между игроками
- [ ] `js/input.js` — разделить локальный и сетевой input
- [ ] `js/dialog.js` — не ставить паузу в кооп
- [ ] `js/save.js` — гость сохраняет прогресс отдельно
- [ ] UI — кнопка "Кооп" в меню, HUD для второго игрока
